## 1. プロジェクト概要

### 1.1 背景
コラボハウスでは、初来場のお客さまに手書きアンケートを記入してもらい、
その内容をスタッフが手作業でANDPAD（施工管理システム）に転記している。
この手作業により以下の問題が発生している：

- 転記ミス（氏名・住所・電話番号の誤入力）
- 転記の遅延（アンケート回収から登録まで数日かかる場合がある）
- 設計士によるヒアリング内容が個人のメモに分散し、共有されない
- ANDPADへの登録漏れ
- ANDPADに登録済みの顧客情報をスタジオ単位で一覧管理する手段がない

### 1.2 目的
1. 手書きアンケートをデジタル化し、回答データがANDPADまで自動連携される仕組みを構築する
2. ANDPADの顧客情報をAppSheetで一元管理し、設計士のヒアリング記録を共有する
3. 転記ミス・遅延・登録漏れをゼロにする

### 1.3 スコープ
| 対象 | 内容 |
|---|---|
| 対象業務 | 初来場アンケート → 顧客情報管理 → ANDPAD連携 |
| 対象拠点 | 全17スタジオ（愛媛・香川・徳島・高知・岡山・広島・大阪・秋田） |
| 対象ユーザー | スタッフ（設計士）、エリアマネジャー、管理者 |
| 対象外 | 2回目以降来場のアンケート、ANDPAD内部の業務フロー |

## 2. 業務フロー

### 2.1 全体フロー

```
[ANDPAD]
  │  毎日6:00 自動エクスポート
  ▼
[共有ドライブ/エクスポートデータ]
  │  GASトリガーで自動取り込み
  ▼
[スプレッドシート/顧客マスタ統合シート] ←── 全件洗い替え
  │
  ├──▶ [AppSheet: 顧客管理アプリ] ←── メインデータソース
  │       │  設計士がヒアリング・加筆
  │       ▼
  │    [スプレッドシート/編集データシート] ←── AppSheet専用データ保存
  │       │  編集された行のみ
  │       ▼
  │    [GAS: ANDPADインポート用xlsx生成]
  │       │
  │       ▼
  │    [共有ドライブ/インポートデータ]
  │       │  andpad-interface で自動アップロード
  │       ▼
  │    [ANDPAD]
  │
  └──▶ [GAS WebApp: 初来場アンケート] ←── STEP0で既存顧客検索にも使用
         │  お客さまがタブレットで回答
         ▼
       [スプレッドシート/Form Responses 2] ←── アンケート専用
         │  日次バッチで編集データシートに転記
         ▼
       [編集データシート] → [ANDPADインポート]
```

### 2.2 データの流れ（3つの経路）

| 経路 | 説明 |
|---|---|
| ANDPAD → AppSheet | 毎日エクスポート → 顧客マスタ統合シート洗い替え → AppSheetで全顧客表示 |
| AppSheet → ANDPAD | 設計士が編集 → 編集データシートに保存 → 編集行のみxlsx生成 → ANDPADインポート |
| アンケート → ANDPAD | 初来場WebApp → Form Responses 2 → 編集データシートに転記 → ANDPADインポート |

### 2.3 お客さまの2パターン

| パターン | 条件 | STEP0での操作 | データの扱い |
|---|---|---|---|
| パターン1：既存顧客 | ANDPADに情報あり | 名前/電話で検索→該当者確認 | インポート時に更新（SystemIDキー） |
| パターン2：新規顧客 | 飛び込み来場 | 「新規」を選択 | インポート時に新規作成 |

> **プリフィル**: 既存顧客選択時に氏名・フリガナ・電話番号をプリフィルする（決定事項D12）。
> 住所・メール等はプリフィルしない。

### 2.4 業務ステップ詳細

#### ステップ0：ANDPAD → スプレッドシート同期（自動）
| 項目 | 内容 |
|---|---|
| トリガー | タスクスケジューラ（毎日6:00） |
| 処理 | ANDPADからエクスポート → 共有ドライブに保存 |
| 後処理 | GASが共有ドライブのxlsxを読み込み → 顧客マスタ統合シートを全件洗い替え |
| 目的 | AppSheetで最新の全顧客を表示 + STEP0の既存顧客検索 |

#### ステップ1：スタッフ記入（STEP0画面）
| 項目 | 内容 |
|---|---|
| 操作者 | スタッフ（受付担当） |
| デバイス | iPad / タブレット |
| 入力項目 | スタジオ名、案内担当者名、来場日、既存/新規選択 |
| 既存顧客の場合 | 名前 or 電話番号で検索 → 選択（氏名・フリガナ・電話をプリフィル） |

#### ステップ2：お客さまアンケート回答（GAS WebApp）
| 項目 | 内容 |
|---|---|
| 操作者 | お客さま |
| デバイス | iPad |
| 画面構成 | STEP1: 個人情報 → STEP2: 家族情報 → STEP3: 住宅計画 |
| 特記事項 | 郵便番号→住所自動入力、フリガナはカタカナ入力 |
| 書き込み先 | Form Responses 2 |

#### ステップ3：AppSheetで顧客管理・ヒアリング
| 項目 | 内容 |
|---|---|
| 操作者 | 設計士 |
| デバイス | iPad（AppSheetアプリ） |
| 表示データ | 顧客マスタ統合シート（ANDPADの全顧客、自スタジオ） |
| 編集項目 | 担当設計士、設計士メモ（5件）、次回来場予定日、予算、土地有無、競合、親承諾、希望入居時期 |
| 保存先 | 編集データシート（SystemIDで紐付け） |
| 完了操作 | 完了フラグをON → ANDPADインポート対象になる |

#### ステップ4：ANDPADインポート（自動）
| 項目 | 内容 |
|---|---|
| トリガー | GASトリガー（毎日0:00） |
| 対象 | 編集データシートの完了フラグONかつANDPAD未連携の行 |
| 処理 | ANDPADフォーマットに変換 → xlsx生成 → 共有ドライブ → andpad-interfaceでアップロード |

## 3. ユースケース

### UC-1：新規のお客さまを受け付ける
| 項目 | 内容 |
|---|---|
| アクター | スタッフ（受付） |
| 正常フロー | 1. STEP0でスタジオ・担当者・来場日入力 → 2.「新規」選択 → 3. アンケート画面へ → 4. iPadをお客さまに渡す |
| 事後条件 | Form Responses 2にデータが記録される |

### UC-2：既存のお客さまを受け付ける
| 項目 | 内容 |
|---|---|
| アクター | スタッフ（受付） |
| 正常フロー | 1. STEP0でスタジオ・担当者・来場日入力 → 2.「既存」選択 → 3. 名前/電話で検索 → 4. 該当者選択（プリフィル）→ 5. アンケート画面へ |
| 事後条件 | Form Responses 2にSystemID付きでデータが記録される |

### UC-3：設計士が全顧客を管理する
| 項目 | 内容 |
|---|---|
| アクター | 設計士 |
| 正常フロー | 1. AppSheetで自スタジオの全顧客一覧を表示 → 2. 対象顧客を選択 → 3. ヒアリング内容を記入 → 4. 保存 |
| 事後条件 | 編集データシートにデータが保存される。顧客マスタ統合シートの洗い替えには影響されない |

### UC-4：ANDPADにデータが自動連携される
| 項目 | 内容 |
|---|---|
| アクター | システム（自動） |
| 事前条件 | 編集データシートに完了フラグONかつ未連携のデータが存在する |
| 正常フロー | 1. GASが編集データシートからxlsx生成 → 2. andpad-interfaceがANDPADにアップロード → 3. 連携ステータス更新 |

### UC-5：クロススタジオで顧客を共有する
| 項目 | 内容 |
|---|---|
| アクター | 管理者 / エリアマネジャー |
| 事前条件 | 対象顧客の編集データが存在する |
| 正常フロー | 1. 顧客詳細画面で「閲覧スタジオを設定」ボタン → 2. 閲覧許可するスタジオをチェック → 3. 保存 |
| 事後条件 | 選択されたスタジオのスタッフもこの顧客をAppSheetで閲覧可能になる |
| 例 | A様が束本で初来場 → 管理者がA様に「高松」を追加 → 高松スタッフもA様を閲覧可能 |

## 4. ロール定義

| ロール | 対象 | 権限 |
|---|---|---|
| スタッフ（設計士） | 各スタジオの設計士 | 自スタジオ（＋追加閲覧スタジオ）のデータ閲覧・加筆 |
| エリアマネジャー | エリア統括 | 担当エリア内の全スタジオのデータ閲覧・加筆 |
| 管理者 | 経営陣・本部 | 全データの閲覧・全項目の編集・削除 |

## 5. 非機能要件（業務観点）

| 項目 | 要件 |
|---|---|
| 可用性 | 営業時間（9:00-18:00）にアンケート入力・AppSheetが利用可能であること |
| データ鮮度 | ANDPADエクスポートは毎日1回（6:00）以上 |
| 連携タイミング | 編集完了からANDPAD登録まで翌営業日以内 |
| セキュリティ | 個人情報を扱うためアクセス権限を適切に制御。プリフィルは氏名・フリガナ・電話番号に限定 |
| 対応デバイス | iPad（主）、スマートフォン（副） |
| データ永続性 | 顧客マスタ統合シートは日次洗い替え。AppSheet専用データ（編集データシート）は永続保持 |

## 6. 前提条件・制約

| 項目 | 内容 |
|---|---|
| ANDPADのAPI | 公開APIなし。HTTPリクエストによるエクスポート/インポート |
| セッション管理 | ANDPADのセッションCookieは約1ヶ月有効。毎日実行で自動延命 |
| Google Workspace | コラボハウスのGoogle Workspaceアカウントが必要 |
| ネットワーク | スタジオにインターネット接続環境が必要 |
| PC常時稼働 | タスクスケジューラ実行のため、1台のPCが常時稼働 |
| Google Form | 廃止。全画面GAS WebAppに移行（決定事項D1） |
| フリガナ | カタカナ入力（決定事項D2） |
