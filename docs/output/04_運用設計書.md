## 1. 日次運用スケジュール

```
00:00  [自動] ANDPADインポート用xlsx生成（GASトリガー）
        └─ 編集データシートから完了フラグON＆未連携データを抽出
        └─ 共有ドライブに xlsx 出力

00:30  [自動] ANDPADインポート実行（タスクスケジューラ → import.js）
        └─ 共有ドライブのxlsx → ANDPADにアップロード → アーカイブ

01:00  [自動] 連携ステータス更新（GASトリガー）
        └─ 編集データシートのANDPAD連携済フラグをTRUEに更新

06:00  [自動] ANDPADエクスポート開始（タスクスケジューラ → export.js）
        └─ エクスポート完了後、共有ドライブに xlsx 保存（最大30分）

07:00  [自動] 顧客マスタ同期（GASトリガー）
        └─ 共有ドライブの最新xlsx → 顧客マスタ統合シート全件洗い替え
        └─ ※ 編集データシートは影響を受けない

07:30  [自動] アンケートデータ転記（GASトリガー）
        └─ Form Responses 2 の未転記データ → 編集データシートにコピー

09:00〜18:00  [手動] 来場アンケート運用 + AppSheet顧客管理
        └─ 初来場: STEP0 → WebApp STEP1〜3 → Form Responses 2
        └─ 顧客管理: AppSheet → 顧客マスタ閲覧 → 編集データシートに加筆
```

## 3. データの永続性

| データ | 洗い替え | 永続性 | 説明 |
|---|---|---|---|
| 顧客マスタ統合シート | 毎日全件 | × | ANDPADの最新データで毎日上書き |
| **編集データシート** | **なし** | **○** | 設計士メモ・予算・土地・競合等は永続保持 |
| Form Responses 2 | なし | ○ | アンケート回答は蓄積（転記済みフラグで管理） |

> **重要**: 顧客マスタ統合シートが洗い替えされても、編集データシートのデータは一切影響を受けない。
> 設計士がAppSheetで入力したメモ・予算・土地有無・競合・親承諾等は、SystemIDで紐付いて残り続ける。

## 4. 障害対応

| # | 障害 | 影響 | 対応 |
|---|---|---|---|
| F1 | ANDPADセッション切れ | エクスポート/インポート停止 | PCで `npm run login` を手動実行 |
| F2 | PC未起動 | J1, J5停止 | PC起動後にStartWhenAvailableで自動実行 |
| F3 | 共有ドライブ接続切れ | xlsx保存/読み込み失敗 | ネットワーク確認 → Gドライブ再マウント |
| F4 | GASトリガー失敗 | J2〜J4, J6停止 | GASエディタで手動再実行 |
| F5 | 郵便番号API障害 | 住所自動入力不可 | お客さまに手入力してもらう |
| F6 | 顧客マスタ洗い替え後にAppSheetで表示されない | SystemID変更 | 編集データシートのRefを再紐付け |

## 5. マスタデータ管理

### 5.1 ユーザーマスタの更新
| イベント | 対応 |
|---|---|
| 新入社員 | ユーザーマスタに行追加 + AppSheetにユーザー追加 |
| 退職・異動 | ユーザーマスタの該当行を削除 or スタジオ名変更 |
| クロススタジオ設定 | 管理者がAppSheet顧客詳細画面で「閲覧スタジオを設定」→ 閲覧許可スタジオを選択 |

### 5.2 スタジオマスタの更新
| イベント | 対応 |
|---|---|
| 新スタジオ開設 | スタジオマスタに行追加 + WebAppのHTMLにoption追加（clasp push） |
| スタジオ名変更 | 全テーブルのスタジオ名を一括変更 |
