#### D1: Google Form廃止 → 全画面GAS WebApp化

- STEP0〜STEP3+完了画面を1つのWebAppで構成
- 理由：郵便番号→住所自動入力、統一デザイン、プリフィル制御の自由度

#### D2: フリガナはカタカナ入力
- ANDPADの「顧客名（カナ）」がカタカナのため、入力時点からカタカナにする
- フォーム上の質問文を「フリガナ（カタカナ）」に変更

#### D3: 郵便番号→住所自動入力
- GAS WebApp内で郵便番号APIを使い、リアルタイムで住所を自動入力する

#### D4: 既存/新規顧客の分岐
- STEP0で「既存のお客さま」/「新規のお客さま」を選択
- 既存の場合：名前 or 電話番号で顧客マスタ統合シートを検索 → 該当者を確認（紐付けキー保持）
- **プリフィルはしない**（個人情報漏洩リスク回避）
- 既存顧客のインポート時は新規作成ではなく更新として扱う

#### D5: データ3分類
- **A. ANDPAD連携対象**: 氏名・カナ・住所・電話・メール・来場きっかけ等（アンケート→ANDPAD）
- **B. 双方向同期**: 担当設計士・次回来場予定日（AppSheet⇔ANDPAD）
- **C. AppSheet専用**: 設計士メモ・予算・土地有無・競合・親承諾・希望入居時期・完了フラグ等（ANDPADに送らない）

#### D6: AppSheet専用カラム定義
- 設計士メモ×5件（各メモ+記入者+記入日のセット）
- 予算（万円）：Number
- 土地有無：Enum（あり/なし/検討中）
- 競合：Text（自由記述）
- 親承諾：Enum（問題なし/懸念あり）
- 親承諾メモ：Text
- 希望入居時期：Text（YYYY-MM形式）

#### D7: ANDPADフィールドマッピング修正
- 反響日：空欄（ANDPAD既存データを維持）
- 初回来場日(実績)：アンケートの来場日を入力
- 反響種別：アンケートの選択肢をそのまま入力（変換しない）
- その他項目:初回来場：スタジオ名をそのまま入力

#### D9: 編集データシート（新規作成）
- AppSheetで加筆した情報は「編集データシート」に保存する（SystemIDで紐付け）
- 顧客マスタ統合シートは毎日洗い替えされるため、AppSheet専用データは別シートに分離
- 編集データシートの内容がANDPADインポートの対象（編集された行のみ）
- 保存項目: 設計士メモ1〜5・予算・土地有無・競合・親承諾・希望入居時期・担当設計士・次回来場予定日・完了フラグ

- AppSheetでは表示しない
- アンケートデータは日次バッチで編集データシートに転記 or ANDPADインポート経由で顧客マスタに反映

#### D11: クロススタジオ対応（修正: D13で方針変更）
- ~~ユーザーマスタに「追加閲覧スタジオ」カラムを追加~~ → D13で廃止

#### D13: クロススタジオ対応（顧客単位）
- **スタッフ単位ではなく、顧客単位で閲覧許可スタジオを設定する**
- 編集データシートに「閲覧許可スタジオ」カラム（EnumList）を追加
- 管理者・エリアマネジャーが特定の顧客に対して、追加で閲覧できるスタジオを設定
- Security Filter: 「担当スタジオ一致 OR 閲覧許可スタジオに自スタジオが含まれる」で判定
- ユーザーマスタの「追加閲覧スタジオ」は削除
- 例: A様が束本で初来場 → 管理者がA様の閲覧許可スタジオに「高松」を追加 → 高松スタッフもA様を閲覧可能

- **変更後**: 編集データシート → 未連携 → 連携待ち → 連携済（3段階遷移）
